# Strange but True

Static GitHub Pages website for **Strange but True**, Luke Hayes' local mix of practical tech help, AI guidance, event/media support, grant and idea development, creative downloads, and early public-infrastructure game design for North Stradbroke Island / Minjerribah.

Live site: https://auraofintelligence.github.io/strange-but-true/

## Current Focus

The site is moving from a simple service brochure into a working public doorway for:

- local tech, AI, device, website and social setup support
- event, projector, media and outdoor cinema support
- grant writing, civic ideas and community project planning
- paid digital music and writing bundles
- the **Community Ledger** onboarding game
- future public profiles, public contribution trails and C-Hour sandbox concepts

## Community Ledger Game

The Community Ledger is a slow, public onboarding path for people who want to support Strange but True's wider creative and civic infrastructure.

Purchases begin in normal AUD fiat checkout. Along with the downloads, buyers can receive a private download link, a temporary password, and an invitation to build a public profile if they want to join the public game.

Both branches are invitations to build a public profile, join the public infrastructure, and start contributing to the narrative:

- **iCi / SOL branch** - the music-story branch. This is unlocked through the qualifying music bundle after a full listen-through. It connects to the I Choose Infinity story signal and future music-linked quests.
- **XRP branch** - the public-ledger branch. This is unlocked through paid writings, supporter bundles or Super Bonus purchases. It connects purchases, public profiles, quests, civic evidence and future contribution trails.

The crypto layer is not the first sale and is not investment advice. It is being framed as a later public ledger, profile and reputation layer for a broader reciprocity-economy experiment.

## License Status

No license file is currently included.

That means the default position is copyright protection: people can view the public repository, but they do not automatically receive permission to reuse the code, writings, music, images, branding or business material beyond what GitHub's own terms allow for viewing and forking.

Recommended approach for this repository:

1. Use a **split license** rather than one blanket license.
2. License the website code under **MIT** if you are comfortable letting others reuse the HTML/CSS/JavaScript structure.
3. Keep all creative content, music, writing, images, strategy, brand assets, business copy and Community Ledger narrative **All Rights Reserved** unless you intentionally release specific items under another license.
4. Consider a Creative Commons license only for selected public documents or samples you genuinely want people to share.

Suggested wording to add later:

```text
Code in this repository is licensed under the MIT License.
All music, writing, images, brand assets, business materials, Community Ledger concepts and narrative content are copyright Luke Hayes / Strange but True unless otherwise stated. All rights reserved.
```

This README is not legal advice. The safest practical next step is to add a `LICENSE-CODE.md` for MIT code use, plus a `CONTENT-LICENSE.md` or README section that clearly reserves rights over the creative and business material.
